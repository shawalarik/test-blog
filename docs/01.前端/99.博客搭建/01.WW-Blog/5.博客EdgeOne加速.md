# 博客EdgeOne加速

## 简介

EdgeOne 全称为 Tencent Cloud EdgeOne，是腾讯云推出的边缘安全加速平台，也是国内首款基于全新架构的真正一体化的边缘安全加速平台1。它集安全防护、网络与应用性能加速、边缘计算和监控运营分析等能力于一体，主要具有以下特点1：

- **全球加速节点覆盖**：基于腾讯云遍布全球的边缘节点，EdgeOne 可让终端在发起请求时，调度至最接近用户的服务节点。其资源遍布全球 70 + 国家与地区，全网带宽 160Tbps+，覆盖全球 2800 + 加速节点，能有效改善跨地区、跨运营商的访问质量问题。
- **动静态内容智能加速**：它可以自动区分网页资源类型，为静态资源提供高速缓存服务，按照缓存规则直接快速响应用户请求；为动态文件基于自研协议优化及回源链路探测选择最佳回源路径访问，能够有效降低用户访问延迟，提高响应速度。
- **丰富的 DNS 能力**：提供权威 DNS 记录解析能力，用户可通过 NS 接入方式，将 DNS 记录一站式托管至 EdgeOne 产品内，享受 DNS 记录解析、DNSSEC、自定义 NS、分权重解析、分线路解析等服务，提升 DNS 解析性能。
- **四层代理加速服务**：基于 TCP/UDP 协议加速服务，通过分布广泛的四层代理节点、独有的 DDoS 防护模块和智能路由技术，实现终端用户就近接入、边缘流量清洗和端口监听转发，为四层应用提供高可用低延迟的 DDoS 防护和四层加速服务。
- **高度可自定义规则**：允许用户在规则引擎内自定义请求配置匹配条件及执行操作，可灵活自定义业务的缓存规则、网络协议、HTTP 头部以及响应内容等，满足不同用户的个性化需求。
- **强大的安全防护**：具备 DDoS 防护、CC 攻击防护、Web 防护、Bot 管理等功能，可识别并拦截 L3/L4/L7 层各类攻击请求，对 DDoS 攻击流量进行清洗，智能 AI 引擎、BOT 策略引擎对 Web、BOT、CC 类型攻击进行行为分析并更新拦截策略，阻断恶意请求到达用户源站，保障业务访问流畅稳定。

## 使用

将域名添加到 EdgeOne 中进行加速，可参考以下操作步骤：

1. **登录控制台并选择站点**：登录[边缘安全加速平台 EO 控制台](https://console.cloud.tencent.com/edgeone)，通过站点列表，选择需配置的站点，进入站点管理二级菜单。
2. **进入域名管理页面**：在左侧导航栏中，单击 “域名服务”>“域名管理”，进入域名管理详情页面。
3. **添加域名**：单击 “添加域名”，填写需添加的加速域名以及对应的源站信息。其中，加速域名用于提供给客户端访问，支持泛域名接入，若接入主域名，填写 “@” 即可。源站可选择 IP / 域名、对象存储源站、源站组三种方式，根据实际情况填写客户端最终访问的资源地址。
4. **选择推荐配置（可选）**：添加域名时，EdgeOne 会根据常见业务场景提供推荐配置，可根据业务场景选择对应的推荐配置，单击 “下一步” 下发配置，也可直接单击 “跳过”，不下发任何配置，进入下一步。
5. **完成域名创建**：配置完成后单击 “下一步”，域名创建完成后，EdgeOne 将为该域名分配一个 CNAME 地址。
6. **配置 CNAME 使加速生效**：根据所选接入模式完成相应操作，使域名加速生效。若为 NS 接入模式，EdgeOne 会自动添加 CNAME 地址，可通过一键添加开启加速；若为 CNAME 接入模式，需自行在 DNS 解析服务商处添加 CNAME 记录；若为 DNSPod 托管接入模式，可通过一键添加由 EdgeOne 自动完成 CNAME 配置，也可单击稍后添加自行配置。

不同接入模式在操作细节上略有差异，具体可参考[快速接入网站安全加速](https://www.tencentcloud.com/document/product/1145/54208?lang=zh)文档。



我的服务器和域名是在阿里云上，所以我选择的方式是`CNAME 接入模式`，只需要在DNS解析中配置相应的设置即可。
